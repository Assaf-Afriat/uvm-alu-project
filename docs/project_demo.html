üíª DUT Overview

The Design Under Test is an 8-bit ALU with valid/ready handshaking, variable latency execution, and a 4-entry output FIFO.

‚öô
ALU Operations
8 Supported Operations
OpcodeOperationLatency
000ADD2 cycles
001SUB2 cycles
010MUL4 cycles
011AND2 cycles
100XOR2 cycles
101SLL (Shift Left)2 cycles
110SRL (Shift Right)2 cycles
111DIV8 cycles
üîå
Interface Signals
Valid/Ready Protocol

Request Interface

  • req_valid - Request valid signal
  • req_ready - DUT ready to accept
  • req_op[2:0] - Operation code
  • req_op1[7:0] - Operand A
  • req_op2[7:0] - Operand B

Response Interface

  • resp_valid - Response valid signal
  • resp_ready - TB ready to accept
  • resp_result[15:0] - Computed result
üí° Key Design Features
  • Variable Latency: Different operations take different clock cycles (2-8 cycles)
  • Output FIFO: 4-entry FIFO buffers results when consumer is slow
  • Backpressure: DUT blocks inputs when FIFO is full
  • Division by Zero: Returns 0xFFFF as error value

üö¶ Backpressure Testing

Random backpressure generation to stress test the DUT's flow control and FIFO mechanisms.

üö¶
Backpressure Configuration
tb_top.sv

Random resp_ready deassertion to simulate slow consumers.

Plusargs

  • +BACKPRESSURE_EN=1 - Enable backpressure
  • +BACKPRESSURE_PCT=30 - 30% chance of deassertion

Percentage Meaning

  • 0% - No backpressure (resp_ready always high)
  • 30% - Light backpressure
  • 50% - Medium backpressure
  • 70%+ - Heavy backpressure (stress test)
