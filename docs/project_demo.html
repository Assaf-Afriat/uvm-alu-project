8-bit ALU

SystemVerilog ALU with valid/ready request and response, variable latency per opcode, a 4-deep output FIFO, and resp_ready backpressure. Closed-loop UVM environment with agent, self-checking scoreboard, eight covergroups, eleven SVA properties, and optional callback hooks — run via scripts/Run/run.py on QuestaSim / ModelSim.

Design specification

Condensed from dut/dut.sv and README.md. The RTL top is alu_dut with an internal fifo_4_deep on the response path.

req_valid/ready + op/op1/op2 ──► alu_dut ──► FIFO ──► resp_valid/ready + 16-bit result

Request / response signals

SignalDirWidthRole
clk, rst_nin1Clock, active-low reset
req_valid, req_readyin / out1Request handshake
req_opin3Opcode (8 operations)
req_op1, req_op2in8Operands
resp_valid, resp_readyout / in1Response handshake (backpressure)
resp_resultout16ALU / divide-by-zero sentinel

Opcode latencies (execution)

Operationreq_opLatency
ADD0002 cycles
SUB0012 cycles
MUL0104 cycles
AND0112 cycles
XOR1002 cycles
SLL1012 cycles
SRL1102 cycles
DIV1118 cycles (op2==0 → 16'hFFFF)

Execution & blocking

  • While executing, the DUT is busy; req_ready is deasserted until the result can be pushed into the FIFO.
  • If the FIFO is full, completion waits — natural coupling with downstream resp_ready.

Virtual interface

  • interface/alu_if.sv bundles all DUT signals for driver and monitor (clocking blocks omitted for tool compatibility as noted in file).

Source of truth: dut/dut.sv.

Full regression verdict (alu_full_regression_test)

After callback stats and protocol checks, report_phase in test/alu_full_regression_test.sv prints a framed summary; on success you should see:

################################################################ ## FULL REGRESSION RESULTS ## ################################################################ PROTOCOL CHECKER: Protocol Errors : 0 ... ## FULL REGRESSION: *** PASSED *** ## ################################################################

Pass requires zero scoreboard compare failures and zero protocol errors from the monitor callback. UVM fatals/errors and SVA still surface through the simulator as usual — aim for a clean log on top of the printed verdict.

Quick run

From README.md. Execute from scripts/Run/.

cd scripts/Run python run.py --test alu_full_regression_test python run.py --test alu_coverage_test --coverage-report python run.py --backpressure 50 python run.py --list

Prerequisites: QuestaSim or ModelSim on PATH, Python 3.x. Flags include --gui, --clean, --seed, --verbosity, --no-backpressure — see run.py --help.
